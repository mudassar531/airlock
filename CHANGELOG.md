## [0.1.0] — 2026-06-15 (release candidate)

Initial release candidate. The full Phase 0–6 build. Self-contained, no
telemetry, no network calls home.

### Added

- **MCP stdio proxy** (`airlock wrap -- <command...>`): byte-faithful
  newline-delimited JSON-RPC relay between an upstream MCP client and a
  downstream MCP server spawned as a child process. Single interceptor seam
  that all later phases extend.
- **Tamper-evident audit log** at `~/.airlock/audit.log`: append-only JSONL,
  sha256 hash chain, ed25519 signing keypair generated by `airlock init`,
  pre-write redaction of secrets (key-name, known service prefixes, and
  high-entropy opaque blobs).
- **Risk classifier**: tool call → `{capability, operation}` from a built-in
  map covering common MCP servers (filesystem, fetch, brave-search, shell,
  git, slack, telegram, vault, stripe), name-token heuristics, and argument
  shape heuristics. Reads of external content are flagged `untrustedRead`.
- **Lethal-trifecta tracker**: per-session state machine that flags any
  outbound action after a private read + an untrusted content read.
- **Risk scoring**: `low | medium | high | critical` with plain-English
  reasons; payments are always `critical`, lethal trifecta is always
  `critical`.
- **Policy engine**: tiny zod-validated YAML DSL with `version`,
  `defaults.on_unmatched`, ordered `rules[]` matching on `capability`,
  `operation`, `risk_min`, `trifecta`, `tool` (literal or trailing-`*` glob);
  deterministic top-to-bottom first-match-wins evaluator; fail-closed loader
  (`--policy` > `./airlock.policy.yaml` > `~/.airlock/policy.yaml` >
  built-in defaults).
- **Default policy pack** (`policy.example.yaml`): never spend money,
  hold the lethal trifecta, confirm destructive filesystem ops, confirm
  shell exec, confirm outbound message send, allow plain reads, ask on
  everything else.
- **Approval channels**: `ApprovalQueue` with default 60s timeout → deny;
  non-blocking `pending` decision protocol so other traffic keeps flowing
  while a human decides; interactive `@inquirer/prompts` CLI prompt on TTY;
  `node-notifier` desktop notification; out-of-band resolution via
  `airlock approve <id>` / `airlock deny <id>` (drops `<id>.resolution`
  file the running proxy watches).
- **CLI**: `airlock wrap`, `init`, `log`, `verify`, `approve <id>`,
  `deny <id>`, `pending`, `policy check`.
- **Streamable HTTP transport**: minimal POST `/mcp` JSON-RPC relay with
  GET `/mcp` SSE stub, bound to 127.0.0.1 by default, `Origin` header
  validated against a localhost allow-list to prevent DNS rebinding.
- **Killer demo** (`npm run demo:unsafe` / `npm run demo:safe`): benign,
  localhost-only prompt-injection scenario that visibly shows the
  unprotected agent leaking a fake secret to a local sink, then Airlock
  holding the same outbound POST as a lethal-trifecta violation.
- **GitHub Actions CI** matrix on Node 20 and 22 running lint → typecheck →
  test → build → CLI smoke.
- **Apache-2.0** license. Open source by design — this is a security tool.

### Security

- All audit log entries are redacted before disk write. A planted fake
  GitHub PAT in `tools/call` arguments never appears verbatim in
  `audit.log` (tested).
- `airlock verify` walks the chain and reports the exact `seq` of the first
  broken entry on tamper, gap, or hash mismatch.
- Invalid policy files cause `airlock wrap` to exit non-zero (3) rather
  than silently allowing.
- Approval timeout defaults to deny — fail closed.
- HTTP transport rejects disallowed `Origin` headers with 403.

### Known limitations

- HTTP transport is POST-only in this release — server-initiated
  notifications over SSE are stubbed; full streaming is follow-up work.
- Approval prompt runs single-file on a serial chain; concurrent asks
  prompt in order, not in parallel.
- The classifier is not exhaustive; unknown tools default to `other/other`
  and hit `on_unmatched: ask` — which is the right safe default but may
  feel noisy on uncommon MCP servers. Add entries to `src/risk/classify.ts`.
- The `wrapTool()` SDK helper for non-MCP agents the spec mentions as a
  stretch goal was not shipped in v0.1.0.
